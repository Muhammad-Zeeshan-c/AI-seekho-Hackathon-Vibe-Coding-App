## 🎯 PROJECT VISION

Build **KaamKaar** — Pakistan's first AI-Agentic home services platform for the informal economy. This is NOT a typical listing app. It is a fully autonomous, reasoning AI agent embedded inside a beautiful Flutter mobile app that can understand user requests in Urdu, Roman Urdu, or English, discover nearby service providers, rank them intelligently, simulate an end-to-end booking, track the worker live on a map as they travel to the client, handle follow-ups, collect feedback, and detect scam/fraud patterns — all orchestrated by a multi-agent AI pipeline powered by Google Antigravity.

Think: **Urban Company × ChatGPT × Google Maps × Uber** — but built for Pakistan's informal economy, with full AI reasoning, explainability, and agentic automation at its core.

## 🤖 AI & AGENTIC SYSTEM ARCHITECTURE

### Google Antigravity — Core Orchestration

Use **Google Vertex AI Agent Builder / Dialogflow CX / Gemini API** as the Antigravity-equivalent orchestration layer. Structure the system as a multi-agent pipeline:

```
User Input (text/voice/image)
         ↓
┌─────────────────────────────┐
│   AGENT 1: INTENT PARSER   │
│   - Language detection      │
│   - Entity extraction       │
│   - Service classification  │
│   Tool: Gemini Pro API      │
└─────────┬───────────────────┘
          ↓
┌─────────────────────────────┐
│ AGENT 2: CONTEXT ENRICHER  │
│   - Geocode location        │
│   - Time normalization      │
│   - User history context    │
│   Tool: Maps Geocoding API  │
└─────────┬───────────────────┘
          ↓
┌─────────────────────────────┐
│ AGENT 3: PROVIDER FINDER   │
│   - Query Firestore DB      │
│   - Filter by availability  │
│   - Distance calculation    │
│   Tool: Maps Distance API   │
└─────────┬───────────────────┘
          ↓
┌─────────────────────────────┐
│ AGENT 4: RANKING ENGINE    │
│   - Score each provider     │
│   - Generate explanation    │
│   - Select top N            │
│   Tool: Custom scoring fn   │
└─────────┬───────────────────┘
          ↓
┌─────────────────────────────┐
│ AGENT 5: BOOKING EXECUTOR  │
│   - Write to Firestore      │
│   - Send FCM notifications  │
│   - Generate confirmation   │
│   Tool: Firebase Admin SDK  │
└─────────┬───────────────────┘
          ↓
┌─────────────────────────────┐
│ AGENT 6: FOLLOW-UP BOT     │
│   - Schedule reminders      │
│   - Status update messages  │
│   - Post-service review ask │
│   Tool: Cloud Scheduler     │
└─────────────────────────────┘
```

Each agent is a Cloud Function that:
1. Receives input + context
2. Uses a tool (API call)
3. Returns structured output + reasoning log entry
4. Passes to next agent

All reasoning is logged to Firestore: `/agent_logs/{booking_id}/steps[]`

### Gemini API Integration

Use `gemini-1.5-pro` or `gemini-2.0-flash` for:

1. **Intent Parsing** — System prompt:
```
You are KaamKaar's intent parser for Pakistan's informal economy.
Extract from user input:
- service_type (from fixed list)
- location (text + attempt geocode)
- datetime (normalize to ISO)
- urgency (urgent/normal/scheduled)
- language (urdu/roman_urdu/english)
- special_requirements (anything extra)
Respond ONLY in this JSON format: {...}
User may write in Urdu, Roman Urdu, or English. Handle all.
```

2. **Provider Bio Generation** — When worker leaves bio blank
3. **Review Summarization** — Summarize all reviews into 2-line summary
4. **Scam Detection** — Analyze patterns and give confidence score
5. **Image Analysis** — Gemini Vision: analyze problem photos and add to booking context
6. **Conversational Follow-up** — Handle "rebook", "change time", "cancel" intents mid-conversation

### Multilingual Support

- Language detection via Gemini (no extra API needed)
- UI strings: use Flutter's `intl` package with `arb` files for Urdu/English
- Urdu font: `NotoNastaliqUrdu` from Google Fonts
- All AI responses match user's detected language
- Roman Urdu is treated as a first-class language (not transliterated away)

double calculateETA(double workerLat, double workerLng, double clientLat, double clientLng) {
  // Haversine formula for distance in km
  double distanceKm = haversine(workerLat, workerLng, clientLat, clientLng);
  // Average speed in Islamabad/Lahore traffic: 20 km/h

  double timeMinutes = timeHours * 60;
  return timeMinutes; // e.g., "~18 minutes"
}

## 🔒 SAFETY & TRUST FEATURES

1. **Worker Verification Tiers:**
   - Unverified (grey badge)
   - Phone Verified (green badge)
   - CNIC Verified (blue badge + label)
   - Platform Vetted (gold badge — after 50+ jobs + 4.5+ rating)

2. **Client Safety:**
   - Share trip/booking with contact (WhatsApp deep link with booking details)
   - Emergency contact can track booking
   - "I feel unsafe" panic button → flags booking, sends alert

3. **Scam Protection:**
   - AI monitors: repeated rescheduling, price inflation, report patterns
   - Workers auto-suspended on multiple serious reports pending review
   - Payment always cash-on-delivery (no upfront payment = less fraud)

4. **Review Authenticity:**
   - Reviews only allowed from completed bookings (client + worker both must confirm completion)
   - AI flags suspicious review patterns (5 stars immediately after registration)
